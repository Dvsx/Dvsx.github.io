源代码里没有做什么过滤，可以通过爆破和sql注入的方法来解，先把代码看懂吧。
isset()函数：在PHP中用来检测变量是否设置，该函数返回的是布尔类型的值，即true/false，
mysql_query()函数：执行一条mysql查询
语法：mysql_query(query,connection)

参数 描述
query 必需。规定要发送的SQL查询。注释：查询字符串不应以分号结束。
connection 可选。规定SQL连接标识符。如果未规定，则使用上一个打开的连接。

$GLOBALS[“_mysql_ston”]这是一个自定义的全局变量，可以不用在意。（ps：MysqlConverter假定此全局变量设置为数据库连接对象，如果转换器找到一个mysql——connect，它将代码转换成为包含$GLOBALS[“__mysqli_ston”]的赋值给mysql_connect函数的结果）

is_object()函数：用于检测变量是否是一个对象，是返回TRUE，否返回FALSE
mysql_error()函数：返回最近调用函数的最后一个错误描述
mysql_fetch_assoc()函数：从结果集中取得一行作为关联数组。返回根据根据从结果集中取得的行生成的关联数组，如果没有更多行，则返回false
mysqli_num_rows() 函数返回结果集中行的数量。

好了，源码分析完了，就很好理解了，对于没有做任何过滤的题目，我们可以通过如下的两种方法来解决。

（1）抓包爆破

说到爆破，就要先了解burp爆破的几个模块

Sniper：单参数爆破，当有多个参数时同一个字典按顺序替换各参数，总之初始值有一个不变
Battering ram：多参数同时爆破，但用的是同一个字典
Pichfork：多参数同时爆破，但用的是不同的字典
Cluster bamb：多参数做笛卡尔乘积模式爆破
