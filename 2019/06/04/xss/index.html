概述

Cross-Site-Scripting简称“CSS”，为了避免与前端叠成样式表的缩写“CSS”冲突，故又称XSS。一般分为反射型，存储型，DOM型。形成XSS的主要原因是程序对输入和输出没有做合适的处理，导致“精心构造”的字符输出在前端时被当作代码解析从而产生危害。

反射型XSS（get）


这是get类型的XSS，只需要输入
<script>alert("XSS")</script>
即可，但是此处限制了输入长度，我们可以在url里输入，来摆脱长度限制

结果成功弹窗

反射型XSS（post）

直接post xss语句就行

结果成功弹窗

存储型XSS

存储型XSS主要指指XSS语句存储在服务器上，如个人信息或发表文章等地方，当用户访问该页面的时候，会触发代码执行，例如此处是一个留言框

我们在留言框里输入XSS语句，就相当于病毒存储在服务器上，当有人访问，就会被触发，从而造成危害

DOM型XSS

DOM型XSS其实是一种特殊的反射型XSS，它是基于DOM文档对象模型的一种漏洞，常见的dom型漏洞是由img标签形成的
<img src=1 onerror=alert(1)>
这就是一条最简单的dom型XSS语句

根据源码，我们可以构造payload(字符类型的记得要加引号)
'> <img src=1 onerror=alert("xss")>

也可以弹窗

DOM型XSS-X

这个XSS类型和上面的差别是，第一种xss是会直接反射，就是会直接弹窗，这一种还需要点击才会触发xss

XSS之过滤

一些网站会对XSS作出一定的防护，例如对script的过滤

这种情况下也可以用dom型的payload来绕过
'> <img src=1 onerror=alert("xss")>

xss之htmlspecialchars


htmlspecialchars() 函数把预定义的字符转换为 HTML 实体。
使用了htmlspecialchars进行处理,是不是就没问题了呢,htmlspecialchars默认不对’处理
构造payload
' onclick='alert(111)'
同样可以弹窗

XSS之href输出


这种类型的xss你上传的内容都会储存在a标签的href属性里面，可以里利用javascript协议来执行js，从而绕过，构造payload
javascript:alert("XSS之href输出")
从而弹窗

XSS之js输出

这种类型的xss和href类型的有什么区别呢？最大的区别就是虽然都存储到了前端，但是位置不同，比如，我们传入参数“111”，会发现参数存储到了这里
只要我们构造出来xss语句，先让上一个script标签闭合，然后构造xss语句即可，payload如下
a' </script><script>alert(111)</script>
就可以触发xss了
